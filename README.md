## Итоговая таблица: JavaScript vs C#

| Аспект | C# | JavaScript |
|--------|-----|-------------|
| **Типизация** | Статическая | Динамическая |
| **Компиляция** | Да (в IL-код) | Нет (интерпретация / JIT) |
| **Точка входа** | `Main()` | Отсутствует |
| **Объявление переменной** | `int x = 5;` | `let x = 5;` |
| **Константы** | `const int X = 5;` | `const X = 5;` |
| **Вывод в консоль** | `Console.WriteLine()` | `console.log()` |
| **Проверка типа** | Во время компиляции | `typeof` |
| **Сравнение** | `==` (по значению, но с учётом типов для значимых) | `==` (нестрогое) / `===` (строгое) |
| **Условные операторы** | `if/else/switch` | `if/else/switch` + тернарный |
| **Циклы** | `for/while/do-while/foreach` | `for/while/do-while/for...of` |
| **Функции** | Методы с явным указанием типов | Функции без типов параметров |
| **Массивы** | `int[] arr = new int[5];` | `let arr = [];` |
| **Объекты** | Требуют определения класса | Можно создать литералом `{}` |
| **ООП** | Классическое (классы, наследование) | Прототипное (классы — синтаксический сахар) |
| **Null-значения** | `null` | `null` и `undefined` |
| **Строгость языка** | Высокая (компилятор не пропустит ошибки) | Низкая (нужна дисциплина разработчика) |

## Главные выводы

1. **JavaScript гибче C#** — можно менять типы переменных во время выполнения, нет строгой компиляции.
2. **JavaScript проще синтаксически** — меньше "церемоний" (не нужно указывать типы, точку входа и т.д.).
3. **JavaScript опаснее** — многие ошибки находятся только во время выполнения, а не на этапе компиляции.
4. **C# безопаснее** — статическая типизация и компилятор предотвращают целый класс ошибок ещё до запуска программы.
